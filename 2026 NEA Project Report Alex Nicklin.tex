\subsection{Problem Description}
A Bulk File Rename Utility is a program that can rename a group of files automatically given a set of rules or patterns that the final file name has to follow.
\begin{figure}[h]
    \centering

    \end{minipage}\label{fig:figure}

It's very easy to see where a Bulk File Rename Utility will come in handy.
A camera will always generate filenames that don't represent the content of the image, so when capturing multiple photos of the same subject, they should be given a consistent filename.
Each photo has a set of technical details stored that describe information about the photo called the metadata.
The metadata stores a Date Taken Original field, that includes a date and time that photo was taken.
When attempting to store photos on file hosting services such as OneDrive or Nextcloud, the photos are ordered by that field.
If the Date Taken Original field isn't correctly assigned, then files on the server will not be correctly ordered.

\subsubsection{Choosing a Programming Language}
To begin this project, I will first be choosing what programming language to write this software in.
Choosing what programming langauge to write this software in is mostly going to be based on my general familiarity with the langauge.
My top languages based on familiarity would be: C++, Java and Python.
Python and Java both compile their source code into bytecode to be executed by a virtual machine.
If I wrote my project using Python or Java, this would require users to download a virtual machine to simply execute my code.
In my opinion, requiring end-users to do more work to run my project is never good
On the other hand, C++ uses ahead-of-time compilation and produces an executable that can run on a chosen operating system, this means I would simply have to execute my final project on Windows and/or Linux and distribute that executable to my end-users.

\subsection{Project Objectives}
\subsubsection{Renaming}
\begin{enumerate}
    \item Change the filename of one file into another filename
    \item Change the filename of a group of files into another filename
\end{enumerate}

\subsubsection{Renaming Chain}
\begin{enumerate}
    \item Develop a group of chains where each adds to the resulting filename
    \begin{enumerate}
        \item Ability to turn off or on each chain
        \item List of Chains:
        \begin{enumerate}
            \item Date
            \begin{enumerate}
                \item Ability to extract the date from the existing filename
                \item Ability to use the date taken set in the metadata
            \end{enumerate}
            \item Seperator
            \begin{enumerate}
                \item Ability to append a group of chars to act as a seperator between chains
            \end{enumerate}
            \item Filename Body
            \begin{enumerate}
                \item Ability to append a specified string
            \end{enumerate}
            \item Numbering
            \begin{enumerate}
                \item Ability to append a number based on a starting value and a padding
                \item Make the number dependent on the number of files selected
            \end{enumerate}
        \end{enumerate}
    \end{enumerate}
    \item Develop a chain editor to change the order and the number of elements in the chain:
    \begin{enumerate}
        \item Ability to add and remove any number of chains from the default value
        \item Ability to change any of the chains
    \end{enumerate}
\end{enumerate}
\subsubsection{GUI}
\begin{enumerate}
    \item Develop a GUI homepage for renaming the files
    \begin{enumerate}
        \item Table showing details of all the files in the current folder:
        \begin{enumerate}
            \item Column's titles should show the current filename, filename to be set, the date taken original and the date taken original to be set
            \item Ability to drag or click to select the files that you want to edit
        \end{enumerate}
        \item General Buttons:
        \begin{enumerate}
            \item Reset to Defaults: Updates the chain back to its default chain
            \item Folder: Ability to change the folder that you are currently working in
            \item Rename Files: After selecting the files you want to edit and editing the parameters of the elements in the chain, then it updates the file names to those set parameters.
        \end{enumerate}
        \item Renaming Chain:
        \begin{enumerate}
            \item Each of the chains should be interactable so that the user can set their parameters
            \item Updates based on changes to the chain editor
        \end{enumerate}
    \end{enumerate}
    \item Develop a GUI for the chain editor
    \begin{enumerate}
        \item Ability to change any of the set default elements to a different chain
        \item Implement Buttons:
        \begin{enumerate}
            \item OK: All changes will be saved to the renaming chain
            \item Reset: All changes will be reset to the default renaming chain
            \item Cancel: All changes made will be removed.
        \end{enumerate}
    \end{enumerate}
\end{enumerate}

\subsection{Description}
This design section aims to create a blueprint for the GUI and code structure of the Bulk Rename Utility.
The main functionality of the program is written in C++ and uses the QT framework to develop the GUIs.

\bigskip

The GUI should produce a list of files in the selected current working directory for the user to interact with.
The user should be able to select which files they'd like to modify by clicking and/or dragging over them in the GUI list.
The user will be able to interact with the GUI to modify each element's parameters in the renaming chain to produce a desired new template for the selected files' names.
The user should also be able to access three separate buttons to reset the renaming chain back to its original chain, change the current working folder and apply all the desired changes and rename the files.
The user, using a separate GUI, will be able to modify the elements in the renaming chain, by adding, removing or changing.

    \item Add a name such as "My Latex Compile"
    \item In the main file to compile, select the .tex file
    \item Leave other settings in place. Click OK
\end{itemize}
